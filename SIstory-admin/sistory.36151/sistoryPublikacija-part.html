26.1. Publikacija

Definicija
Digitalne datoteke objavljenih objektov, skupaj z njihovimi metapodatki.
XML element
PUBLICATION
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba:
  • Najprej se naloži datoteka. Na prvem mestu naj bo vedno PDF datoteka. Če se naloži PDF, se pojavi okence za vpisovanje strani, na kateri se začne child (članek, poglavje ipd.) naloženega PDF-ja. Če se številko strani ne vpiše, se opisovani objekt privzeto začne na prvi strani.
  • XML datoteka se zgenerira avtomatično po indeksaciji PDF datoteke (pri kateri je bil narejen OCR). Če naložimo novo XML datoteko se pojavi okence, kjer izberemo XML tip te datoteke. Če izberemo "Navaden" tip, bo XML datoteka samo shranjena in je ne bo mogoče videti na spletni strani portala SIstory. Če pa naložimo TEI XML datoteko, zberemo "Tei" tip. Slednji je prikazan na spletni strani portala SIstory. Pri tem lahko naložimo več TEI XML datotek, prikazana pa bo vedno samo tista, ki bo najvišje na seznamu naloženih datotek. Vrstni red naloženih datotek spreminjamo s puščicami.
  • Na spletni strani portala SIstory so prikazane samo JPEG slike. Lahko se naloži tudi PNG slike, ki se pri prikazovanju prekonvertirajo v JPEG. TIFF slike se lahko samo naloži zaradi arhiviranja, ne bodo pa upoštevane pri prikazovanju.
Atributi datoteke
Vsaka digitalna datoteka ima lahko zabeležene svoje metapodatke:
  • digitalMeasurement
  • digitalFormat
  • digitalDate
  • digitalDescription
  • digitalRights

26.2. digitalMeasurement

Definicija
Obseg in velikost digitalne datoteke.
Izvorni element
vracore:vra/image/measurementsSet/measurements
XML element
PUBLICATION/ATTRIBUTE[@Type='digitalMeasurement']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
Ne vpisuje se za PDF, saj se pri njem avtomatično zgenerira.

26.3. digitalFormat

Definicija
Format digitalne datoteke.
XML element
PUBLICATION/ATTRIBUTE[@Type='digitalFormat']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
  • Ker se format datoteke v sistemu SIstory avtomatično zgenerira iz končnice imena datoteke (npr. ime_datote.pdf je PDF datoteka) oziroma se razlika med XML dokumentom in XML TEI označi posebej, običajno ni potrebno zapisovati tega podatka.
  • Ta podatek se zapiše samo v primeru, če naložimo datoteko formata, ki jo ne podpira sistem SIstory. Pri tem lahko ta podatek zapišemo:
    • kot splošen opis
    • ali pa bolj natančno kot MIME Media Type - IMT.

26.4. digitalDate

Definicija
Datum nastanka digitalne datoteke.
Izvorni element
vracore:vra/image/
XML element
PUBLICATION/ATTRIBUTE[@Type='digitalDate']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
Priporočljivo je zapisati datum nastanka digitalne datoteke. Tega je mogoče zapisati samo kot W3CDTF (brez časa).

26.5. digitalDescription

Definicija
Opis vsebine, nastanka, pridobitve ipd. digitalne datoteke.
Izvorni element
vracore:vra/image/descriptionSet/description
XML element
PUBLICATION/ATTRIBUTE[@Type='digitalDescription']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
Navajanje kakršnega koli drugega metapodatka o digitalni datoteki. Navaja se lahko v povsem opisni obliki.

26.6. digitalRights

Definicija
Avtorska pravica v povezavi z digitalno datoteko.
Izvorni element
vracore:vra/image/rightsSet/rights
XML element
PUBLICATION/ATTRIBUTE[@Type='digitalRights']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
Navedba podatkov o morebirnih avtorskih pravicah digitalne datoteke, ki so drugačne od avtorskih pravic izvornega analognega gradiva.
