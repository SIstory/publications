15. Identifier

>>

<<

>>

1V sklopu Identifier se lahko izbere tri vrste identifikatorjev:

  • SIstory signatura
  • Identifier
  • Bibliographic Citation

15.1. SIstory signatura

Definicija
Signatura analognega arhivskega gradiva.
Element
sistory:signature
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)

15.2. Identifier

Definicija
Alternativna identifikacijska oznaka
Izvorni element
dc:identifier
Originalna definicija
"An unambiguous reference to the resource within a given context. Recommended best practice is to identify the resource by means of a string conforming to a formal identification system.".
XML element
IDENTIFIER[@identifierType='Identifier']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
  • Ožji pomen od dc:identifier.
  • Glavna identifikacijska oznaka publikacij na portalu SIstory se avtomatično generira (kot auto-increment) in je prisotna v URN te publikacije.
  • Identifikacijska oznaka originalnega analognega arhivskega gradiva se vpisuje v polje Sistory signatura.
  • V polje Identifier se vpisuje vse morebitne ostale identifikacijske oznake.
Primeri
V primeru splošno standardiziranih identifikacijskih označb se jih zapisuje na način, da se črkovna označba loči od številčne s podpičjem, npr.:
  • COBISS.SI-ID:236081920
  • ISBN:978-961-6386-12-8
  • doi:10.1000/1

15.3. Bibliographic Citation

Definicija
Bibliografska navedba vira gradiva.
Izvorni element
dcterms:bibliographicCitation
Originalna definicija
"A bibliographic reference for the resource. Recommended practice is to include sufficient bibliographic detail to identify the resource as unambiguously as possible."
XML element
IDENTIFIER[@identifierType='Bibliographic citation']
Tip vnosa
Opcijski (Minimalno: 0, Maksimalno: neomejeno)
Uporaba
  • Natančna bibliografska navedba vira objavljenega gradiva. Npr. vir članka je revija, v kateri je ta članek izšel. Pri tem se vedno navede popoln citat revije, npr.:
    • Naslov revije, leto 5, 2013, št. 3, str. 15-30
  • Ožji pomen od dcterms:bibliographicCitation.
  • V primeru, da je na portalu SIstory npr. objavljena celotna revija in ne le posamezni članek, so bibliografski podatki o reviji shranjeni v relacijski povezavi child(članek)-parent(revija). SIstory URN parenta je razvidna v polju Vir - URN Naslov.
  • V primeru, da za vir gradiva ne obstaja popolni bibliografski zapis, se podatke o njem navede v polje isPartOf, kjer se ga pri navedbi atributa jezika poljubno opiše, ali pa se ga označi z URL oziroma URI naslovom povezave do gradiva izven portala SIstory.
